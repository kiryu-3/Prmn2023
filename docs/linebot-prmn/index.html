LINE Developersコンソールにログインする

それぞれのLINEのアカウントでログインしてください。

開発者として登録する（初回ログイン時のみ）

それぞれのニックネームやメールアドレスを入力してください。

学生の方は自身の大学のメールアドレスを推奨します。

新規プロバイダーを作成する

任意の（好きな）名前を入力してください。

チャネルを作成する

  • Channel Type（チャンネルの種類）が Messaging API になっていることを確認してください。
  • 任意 と書かれている項目は空欄にしてください。
  • チャンネル名や説明は好きなものを入力してください（個人情報ではないもの）。
  • 大業種は個人、小業種は個人（学生）など自分の立場に合わせて選んでください。
  • アイコン画像も自分の好きなものを利用してください。

チャネルを作成する

  • Channel Type（チャンネルの種類）が Messaging API になっていることを確認してください。
  • 任意 と書かれている項目は空欄にしてください。
  • チャンネル名や説明は好きなものを入力してください（個人情報ではないもの）。
  • 大業種は個人、小業種は個人（学生）など自分の立場に合わせて選んでください。
  • アイコン画像も自分の好きなものを利用してください。

LINE Developersコンソール のプロバイダーの欄から 、

あなたが作成したプロパイダー名 → あなたが作成したチャンネル名 の順に選択してください。

チャンネルを開き、Messaging API設定 のタブを開いてください。

・ WebHookURL を、セルに表示されているhttps://xx-xx-xx-xx-xx..ngrok.io/callback に変更してください。

（ここについては参考資料のノートブック内で説明があるはずですので、今は無視していただいて結構です）

・ Webhookの利用 を、選択状態（緑色）にスイッチしてください。

（WebHookURL指定後の話になります）

・ 応答メッセージの 編集 を選ぶと、応答設定のウィンドウが開くはずです。

  • 応答メッセージ を オフ にする
  • あいさつメッセージ を オフ にする
  • Webhook は、 オン のまま にする（間違えないように！）
  • ここまで設定したら、応答設定 のウィンドウを閉じる。

・ チャネルアクセストークン（長期） の 発行 ボタンを押してください。

WebHookURLはセルを実行するたび毎回変化します。

ブラウザのタブなどで開いたままにしておくと便利です。
